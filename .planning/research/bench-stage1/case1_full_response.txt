파일을 직접 확인하기 전에, 컴포넌트 이름과 일반적인 React 패턴 기반으로 원인을 먼저 추정해볼게요.

## 원인 추정

timezone 변경 후 DatePicker가 즉시 반영 안 되는 문제는 거의 확실하게 다음 3가지 중 하나(또는 복합)입니다:

### 1. Date 객체 초기화 시점에 timezone이 캡처되고 이후 갱신 안 됨

가장 흔한 패턴입니다. DatePicker 내부에서 `useState`나 `useRef`로 Date 객체를 초기화할 때 현재 timezone을 기반으로 변환해놓고, `TzContext`가 바뀌어도 이미 만들어진 Date 인스턴스를 재계산하지 않는 경우입니다.

```tsx
// 전형적인 버그 패턴
const { timezone } = useTzContext();
const [selectedDate, setSelectedDate] = useState(() =>
  convertToTz(new Date(), timezone)  // ← 최초 렌더 시점의 timezone으로 고정
);
```

`timezone`이 바뀌어도 `useState`의 initializer는 다시 실행되지 않으니까 `selectedDate`는 옛날 timezone 기준으로 남아있게 됩니다.

### 2. useMemo/useCallback의 deps에 timezone 누락

DatePicker가 내부적으로 날짜 포맷팅이나 캘린더 그리드 계산을 `useMemo`로 하고 있는데, dependency array에 `timezone`을 안 넣었을 가능성이 높습니다.

```tsx
// 버그
const formattedDates = useMemo(() => {
  return generateCalendarGrid(currentMonth, timezone);
}, [currentMonth]);  // ← timezone 빠짐

// 수정
}, [currentMonth, timezone]);
```

### 3. 외부 date library 인스턴스가 timezone을 전역/싱글턴으로 관리

만약 `dayjs`, `luxon`, `date-fns-tz` 같은 라이브러리를 쓰고 있고, 초기화 시점에 timezone을 설정하는 방식이라면 context가 바뀌어도 라이브러리 내부 설정이 갱신 안 될 수 있습니다. 예를 들어 `dayjs.tz.setDefault(timezone)`을 앱 시작 시 한 번만 호출하는 경우.

## Fix 방향

**단기 (즉시 적용 가능)**

1. **DatePicker 내부에서 `timezone`을 `useEffect` deps에 포함시키기** — timezone 변경 시 내부 Date 상태를 재계산하는 effect 추가:
```tsx
const { timezone } = useTzContext();

useEffect(() => {
  setSelectedDate(prev => reconvertToTz(prev, timezone));
  // 캘린더 그리드도 재생성
}, [timezone]);
```

2. **모든 useMemo/useCallback deps에 timezone 추가** — lint rule로 강제하면 좋음

3. **Date 포맷팅을 렌더 시점에 수행** — 저장은 UTC, 표시만 timezone 적용. 이러면 timezone 변경 시 리렌더만으로 자동 반영됨:
```tsx
// 상태는 항상 UTC로
const [selectedDate, setSelectedDate] = useState<Date>(new Date());

// 표시는 렌더 시점에 변환
const displayDate = useMemo(
  () => formatInTimezone(selectedDate, timezone),
  [selectedDate, timezone]
);
```

**중기 (구조 개선)**

4. **TzContext에 `key` prop 패턴 적용** — timezone 변경 시 DatePicker를 완전히 remount:
```tsx
<DatePicker key={timezone} />
```
이건 quick fix로는 좋지만, 사용자가 입력 중이던 상태가 날아갈 수 있어서 UX 검토 필요.

5. **date utility layer 통일** — timezone 변환 로직이 DatePicker 내부에 흩어져 있으면 `useTzDate` 같은 커스텀 훅으로 추출해서, timezone 변경 → 자동 재계산 경로를 한 곳에서 보장.
