**결론: 외부 트리거로 in-memory clear 불가능.**

발견:
- **`/clear`**: 정확히 그 동작 ("wipes conversation but keeps files") — but **slash command 만**, hook/MCP/signal 외부 트리거 경로 0
- **Hook output schema 전체**: `additionalContext`, `decision:block`, `updatedInput`, `updatedMCPToolOutput` — 메모리 mutate 필드 **없음**
- **Signal**: SIGUSR 핸들러 없음 (SIGCONT/HUP/INT/TERM/WINCH만)
- **MCP `_meta`**: progress/task tracking 전용, context 조작 불가
- **Slash command 핸들러**: React useRef (`LHH.current`) — 외부 접근 불가

**Cache 보존 한계**: 메시지 배열을 비우면 그 부분 prefix 사라짐 → 무조건 cache miss. system prompt + tools 부분만 cache hit 보존 가능.

- `autoCompactEnabled`, `autoCompactWindow`, `autoCompactThreshold` (settings.json)
