<objective>
Build the reproducible leakage-check fixture that closes D-15 gate item #3: a 5-sub-agent Claude Code interactive test whose canary tokens never appear in the main conversation transcript. This plan creates the fixture-prep helper, the grep-based leakage check script, and the fixture markdown file — then runs them live against a user-driven fresh CC session.

Purpose: CAP-03 is the strongest empirical claim the hypothesis makes — "5 concurrent sub-agents leave 0 body text in the main transcript". Without a reproducible fixture anchored to specific canary tokens and a specific session transcript path, the claim is unfalsifiable. Research finding 1 + finding 2 give us everything we need: the canonical transcript path (`~/.claude/projects/<slug>/<sessionId>.jsonl`), the sessions index (`~/.claude/sessions/*.json`), and the knowledge that sub-agents sharing the parent MCP connection all exercise the same `handleWriteReport` path.

Output:
- `scripts/leakage-fixture-prep.ts` — canary generator + fixture prompt printer
- `scripts/leakage-check-interactive.ts` — transcript discovery + grep + exit-code report
- `.planning/phases/01-stripping-path-interactive-mode-only/01-LEAKAGE-CHECK.md` — fixture text + post-run receipt
</objective>

       - Resolves transcript path: `join(homedir(), ".claude", "projects", slug(cwd), `${session.sessionId}.jsonl`)`.
       - Resolves subagents dir: `join(homedir(), ".claude", "projects", slug(cwd), session.sessionId, "subagents")`.
       - For each canary in the input file:
         * Count occurrences in the main transcript via `readFileSync(path, "utf8").split(canary).length - 1`.
         * If subagents dir exists: iterate every `*.jsonl` file, sum canary occurrences across all.
         * Count occurrences in `.sagol/reports/*.md` by reading each file under `PROJECT_ROOT/.sagol/reports/`.
         * Collect into a row: `{ canary, mainTranscript, subagents, disk }`.
       - For `tool_use.input` classification (the "unavoidable input leak" per research pitfall): naive string grep in the main transcript is already the right check. The research notes that for sub-agent-mediated writes, main-transcript tool_use blocks only contain the Task() description, not the report body — so the expectation is `mainTranscript === 0` even for tool_use.input. The script does NOT need to JSON-parse to classify — it just asserts main-transcript hit count is 0 and labels any non-zero count as FAIL with the advisory "if this is `tool_use.input`, the main agent passed the body directly — sub-agents were bypassed".
       - Print a table: `canary | main | subagents | disk`.
       - Exit 0 iff every `mainTranscript === 0` AND every `disk === 1` AND total distinct report IDs seen is exactly 5.
       - Exit 1 otherwise with a clear RED summary.
       - If `findNewestInteractiveSession` returns `null` OR the transcript file does not exist, exit 2 with message "no interactive session found for cwd=<cwd>. Did you run the fixture in this project's Claude Code session?".
       - Print sessionId, startedAt (as human-readable), transcript path, subagents dir (exists/missing).

Exit code 1 is expected at this stage because disk count is not 5. This confirms the discovery + grep plumbing works. Capture the output for the summary.
