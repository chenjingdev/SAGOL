       - Shebang `#!/usr/bin/env bun`
       - Header comment explaining: "Idempotent init — merges `.mcp.json` and `.claude/settings.json` under cwd. NEVER touches `~/.claude/settings.json`. NEVER writes the dormant `PostToolUse` hook block on virgin installs (see D-10 rationale)."
       - Parse `process.argv` for `init` subcommand and `--dry-run` flag. The script's first positional after `bun run` / bin invocation is the subcommand.
       - If invoked as `sagol init` (or just `init`): perform the merge for `.mcp.json` and `.claude/settings.json`.

       - Helper `readJsonOrEmpty(path, fallback)`: `Bun.file(path).exists()` check, then `JSON.parse(await Bun.file(path).text())` with try/catch (fail loudly on invalid JSON with message "bunx sagol init: <path> is not valid JSON. Fix it manually first and re-run.").
       - Helper `writeJsonPreserving(path, obj, dryRun)`: `JSON.stringify(obj, null, 2) + "\n"`; if `dryRun` print `--- <path> ---\n<content>`, else `await Bun.write(path, content)`.
       - Main `init()` body:
         a. Resolve CWD.
         b. Read `.mcp.json` or use `{}`. Ensure `mcpServers` object. Set `mcpServers.sagol = { command: "bun", args: ["run", "src/mcp/server.ts"] }`.
         c. Read `.claude/settings.json` or use `{ $schema: "https://json.schemastore.org/claude-code-settings.json", enabledMcpjsonServers: [] }`. Ensure `enabledMcpjsonServers` is an array; if missing/non-array → initialize to `[]`. If not including `"sagol"` → push `"sagol"`. DO NOT read, inspect, or write the `hooks` block on virgin files.          d. On `--dry-run`: print both merged objects to stdout under `--- .mcp.json ---` / `--- .claude/settings.json ---` headers, exit 0 without writing.
         e. Otherwise: compute the current on-disk content (if any) and the target content. If they are byte-equal → print `sagol: .mcp.json already up to date` / ditto for settings and do NOT call `Bun.write` (true idempotency). Otherwise write.
         f. Print a final `sagol init complete. Next: bunx sagol doctor` line.
